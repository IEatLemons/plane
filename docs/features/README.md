## 功能记录目录说明（docs/features）

这个目录用于**按功能模块拆分**存放所有功能开发记录，由 AI 助手在协作过程中自动维护。

### 推荐目录结构

- `workspace/`：工作区、成员、邀请、权限相关功能
- `project/`：项目、任务、迭代、视图等核心工作项相关功能
- `inbox/`：收件箱、通知、分派相关功能
- `onboarding/`：新手引导、创建团队、邀请成员等首次使用流程
- `settings/`：个人设置、工作区设置、系统设置相关功能
- `integration/`：第三方集成（如 GitHub、Slack 等）相关功能
- `others/`：暂时无法归类或跨模块的功能，后续可以再拆分

你在对话中提到每一个新功能时，我会：

1. 根据你的描述判断所属模块（如有歧义会在记录里注明假设），在对应子目录下创建功能文件。
2. 使用统一模板写入：功能目标、范围、优先级、当前状态、进度时间线等信息。
3. 同步更新 `docs/feature-development-tracking.md` 中的总览索引，方便你从一个地方跳转到所有功能。

如果你有更偏好的模块划分方式（例如完全按照业务域重新命名目录），也可以直接告诉我，我会按你的方案调整现有结构与后续记录规则。
