## Plane 前端与协作服务概览

本仓库中与前端和实时协作相关的主要应用包括：

- `apps/web`
- `apps/admin`
- `apps/space`
- `apps/live`

它们分别承担不同角色，组合在一起构成 Plane 的产品体验。

---

## web：主产品 Web 应用

- **路径与端口**
  - 开发：`pnpm --filter=web dev`，默认监听 `3000`
  - Docker 本地：服务名 `web`，端口映射 `3000:3000`
- **目标用户**
  - 普通工作区成员、项目协作用户
- **核心职责**
  - 工作区、项目、任务（issue）等核心业务的主界面
  - 看板、列表、甘特等视图的展示与交互
  - 利用 `@plane/editor` 提供富文本编辑体验
  - 利用 `@plane/propel`、`@plane/ui` 提供报表与可视化组件
  - 通过 `@plane/services` 调用后端 API，与 `apps/api` 协作完成业务逻辑
  - 支持国际化、深浅色主题等通用产品能力
- **在整体架构中的位置**
  - 整个产品的「主入口」，绝大多数用户日常只会直接接触 `web`
  - 与 `live` 协作提供多人实时编辑，与 `api` 协作提供数据读写与权限控制

---

## admin：系统与组织管理后台

- **路径与端口**
  - 开发：`pnpm --filter=admin dev`，默认监听 `3001`
  - 本地 Docker：服务名 `admin`，端口映射 `3001:3001`
  - 默认基础路径：`VITE_ADMIN_BASE_PATH="/god-mode"`，即通常通过 `/god-mode/` 访问
- **目标用户**
  - 系统管理员、组织 Owner、运维与 Support 人员
- **核心职责**
  - 系统级 / 组织级配置管理（如多租户、配额、集成开关等）
  - 用户、角色与权限的管理界面
  - 使用 `@plane/propel` 和 `@plane/ui` 构建的监控、统计、报表视图
  - 一些不适合暴露在普通用户界面的高级维护能力
- **在整体架构中的位置**
  - 站点的「控制塔」，决定全局行为和安全边界
  - 依然复用 `@plane` 系列 workspace 包，与 `web` 共用很多基础设施，但路由与入口完全独立

---

## space：公共 / 分享空间应用

- **路径与端口**
  - 开发：`pnpm --filter=space dev`，默认监听 `3002`
  - 本地 Docker：服务名 `space`，端口映射 `3002:3002`
  - 默认基础路径：`VITE_SPACE_BASE_PATH="/spaces"`，访问路径通常为 `/spaces/...`
- **目标用户**
  - 访客用户、通过分享链接进入的外部协作者
  - 需要浏览公开空间 / 项目的人
- **核心职责**
  - 承载公开空间、公开项目等对外展示页面
  - 提供权限受限 / 只读视图，和内部 `web` 视图区分开
  - 使用 React Router + SSR（`@react-router/serve`）以获得更好的首屏和 SEO
- **在整体架构中的位置**
  - 作为「公共 Portal」，将内部空间通过可控的方式暴露给外部
  - 与 `web` 共享类型、UI、服务调用，但在访问路径和权限模型上更偏「公开展示层」

---

## live：富文本与文档的实时协作服务

- **路径与端口**
  - 开发：`pnpm --filter=live dev`，在 Node 中运行，默认端口在 `.env` 中配置（本地 Docker dev 映射为 `3003:3003`）
  - 在 `docker-compose-local.yml` 中服务名为 `live`
- **目标用户**
  - 终端用户不会直接访问界面；这是被 `web` 等前端消费的后端服务
- **核心职责**
  - 提供基于 `@hocuspocus/server` 与 `yjs` 的实时协作内核
  - 维护多人同时编辑文档 / 富文本时的会话、冲突合并与同步
  - 暴露 WebSocket 与 HTTP 接口，供前端连接和心跳检测
  - 通过 Redis 等后端组件持久化协作状态与会话信息
- **在整体架构中的位置**
  - 相当于「协作引擎」：为 `web` 和未来可能的其他客户端提供实时同步能力
  - 与 `api` 协调：`api` 负责持久化业务数据与规则，`live` 专注于实时编辑和短期状态管理

---

## 典型访问路径总结

- 本地开发直连（不经过 proxy）：
  - Web 主站：`http://localhost:3000/`
  - 管理后台：`http://localhost:3001/god-mode/`
  - 公共空间：`http://localhost:3002/spaces/...`
  - 实时协作服务：由前端通过 WebSocket / HTTP 调用，例如 `live` 服务配置的端口与路径

在生产环境中，通常会通过反向代理（如 `apps/proxy`）将这些服务统一到同一个域名下的不同路径，例如：

- `/` → `web`
- `/god-mode` → `admin`
- `/spaces` → `space`
- `/live` 或某个子域 → `live`

实际路由取决于部署配置，但职责划分与上文一致。
