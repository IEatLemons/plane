## 功能开发记录与进度总览

本文件由 AI 助手维护，用于在协作过程中记录你提出的每一项功能需求及其开发进度，并作为**按功能模块拆分文档的索引入口**，便于多次会话之间保持上下文与一致性。

### 结构与存放位置

- **根目录**：所有具体功能记录文件统一放在 `docs/features/` 目录下。
- **模块划分**：根据功能模块再分子目录，例如：
  - `docs/features/workspace/`
  - `docs/features/inbox/`
  - `docs/features/onboarding/`
  - `docs/features/permissions/`
  - 后续根据需要再补充新的模块目录。
- **单功能文件**：每个功能一个独立 Markdown 文件，命名建议：
  - `<模块>-<简要功能名>.md`，例如：`workspace-invite-members-flow.md`
  - 或者使用 ID：`FEATURE-123-workspace-invite-members.md`

> 实际具体模块划分，我会结合你在对话中对功能的描述来选择或新增子目录。

### 记录方式与约定

- **记录范围**：你在对话中说明的每一个独立功能（无论大小），都会在 `docs/features/**` 下单独建一条记录。
- **更新频率**：每当你补充需求、进度发生变化，或我完成了某一步实现时，我都会同步更新对应的功能文件。
- **总览索引**：本文件会维护一个「索引列表」，方便快速跳转到各个模块/功能。
- **排序方式**：默认按创建时间倒序（最近的在最上方），必要时可以按模块或优先级重排。

---

### 统一模板（示例）

- **功能名称**：示例功能
- **标识 ID**：FEATURE-EXAMPLE
- **相关范围**：apps/web, apps/api
- **当前状态**：规划中 / 开发中 / 待联调 / 待测试 / 已完成 / 暂停
- **优先级**：高 / 中 / 低
- **简要描述**：
  - 描述功能的目标、核心用户场景和关键约束。
- **当前进度**：
  - 2026-03-16：创建记录。
- **备注**：
  - 需要时补充设计链接、接口文档等。
